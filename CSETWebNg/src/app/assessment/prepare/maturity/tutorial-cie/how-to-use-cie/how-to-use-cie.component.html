Identifying the Scope of the Analysis

Identifying the scope of the organization’s analysis is important because answers to the Implementation Guide questions in this CIE Analysis tool must be provided in relation to a specific scope. Users can respond to questions under two different scopes for each Principle. Each principle is independent, and the user can decide the direction for each principle or choose to do both scopes for every principle. The two scopes are:

  1. Principle scope: Answer high-level questions for each of the 12 Principles only.
  2. Principle-Phase scope: Answer detailed questions for each principle throughout each phase of the System Engineering Lifecycle Model. Within each of the 12 Principles, a user will respond to questions under 7 lifecycle phases and 1 set of cross-cutting considerations.
